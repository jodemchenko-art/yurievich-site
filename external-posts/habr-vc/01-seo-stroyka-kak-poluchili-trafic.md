## Проблема: за 6 лет в стройке мы получали 80% клиентов с Авито

К началу 2026 наша воронка выглядела так:

| Канал | Доля заявок | Стоимость заявки |
|---|---|---|
| Авито (платный тариф «Максимальный») | ~70% | 450 ₽ |
| Сарафан (рекомендации) | ~20% | 0 ₽ |
| Google/Яндекс органика (старый сайт-визитка) | ~10% | 0 ₽ |

Бюджет на Авито рос: январь 2025 — 180к, январь 2026 — 350к. Цена заявки выросла с 280 до 450 рублей. На декабрь 2025 уже шёл к 600 рублям из-за сезонного демпинга.

Главное: эти заявки **сезонные**. Зима — поток падает в 3 раза. Дачники строят весной.

Я хотел канал, который:
1. Не зависит от Авито и его тарифной политики
2. Работает круглый год (а не вспышками сезона)
3. Не требует ежедневной возни (запустил → работает)
4. Имеет потолок выше 100 заявок в месяц (Авито давал максимум 60)

Очевидный ответ — SEO.

## Что сделали (за 3 недели)

### 1. Переписали на Next.js 14 SSG

Тильда хороша для лендинга, но не для контентной стратегии. Дублирование контента, медленные страницы, ограничения в SEO-разметке.

Next.js даёт:
- 100% статическую генерацию (SSG) для всех страниц — никаких серверных задержек
- Полный контроль над head, meta, structured data
- Edge-функции для API (`/api/lead`, `/api/notify`, `/api/indexnow`) без своего сервера
- Хостинг на Vercel бесплатно (Hobby plan)

Стек:
- Next.js 14 App Router
- TypeScript (типизация контента)
- Tailwind CSS (стилинг)
- Vercel deploy + edge
- GitHub Actions (CI)

### 2. Structured data — это критично, особенно в Яндексе

Самый недооценённый аспект SEO в РФ. Яндекс **сильно** учитывает Schema.org для:
- Расширенных сниппетов в выдаче (звёзды, цена «от», крошки)
- Попадания в Алису/Нейро как источник
- Entity-based ranking (связывание сущностей через @id)

Что мы сделали:
- **Единый @graph JSON-LD на каждой странице** (~1 KB JSON, 12-15 сущностей)
- Organization + 3 Person (Юрий + Валера + Евгений) + WebSite + 3 Service в siteGraph
- Article (на блог-страницах) + FAQPage + BreadcrumbList — связаны @id-ссылкой на Person Юрий как автора
- Service (на гео-страницах) + AreaServed + Offer + FAQPage

Код вытащен в `lib/schema.ts`:

```typescript
export function buildSiteEntities() {
  return [
    {
      '@type': 'GeneralContractor',
      '@id': ID.org,
      name: SITE.legalName,
      // ... PostalAddress, GeoCoordinates, employee[], makesOffer[]
      founder: { '@id': ID.yuri },
      employee: [{ '@id': ID.yuri }, { '@id': ID.valery }, { '@id': ID.evgeny }],
    },
    // 3 Person entities со своими knowsAbout, jobTitle
    // WebSite
    // 3 Service со своими offers
  ];
}

export function buildArticleGraph(article, canonicalUrl, breadcrumbPath) {
  return [
    {
      '@type': 'Article',
      '@id': ID.article(article.slug),
      author: { '@id': ID.yuri }, // E-E-A-T через Person а не Organization
      publisher: { '@id': ID.org },
      // ...
    },
    buildBreadcrumb(breadcrumbPath, [...]),
    buildFaqPage(breadcrumbPath, article.faq),
  ];
}
```

Идея: каждая сущность имеет стабильный `@id`. Ссылки между ними — через `@id`-ссылку, не дубликаты. Это сильно помогает Яндексу строить «карту бизнеса».

### 3. Динамические гео-посадки

9 страниц вида `/fundament/[район]/`:
- `lib/regions.ts` — 9 регионов с уникальными grountDescription, examples, drivingTime, defaultGround
- `app/fundament/[region]/page.tsx` — `generateStaticParams` от REGIONS
- Уникальный H1, meta, Schema.org Service + areaServed + AdministrativeArea
- Калькулятор с предзаполненным грунтом района
- 3 реальных кейса с локацией и ценой по нашим объектам

Это закрывает запросы типа «плитный фундамент в Гатчине цена» — 50-200 показов/мес каждый.

### 4. Конвейер автоматической публикации статей

Главная фишка — мы **не нанимаем копирайтеров**. Все статьи генерирует claude.ai Routine ежедневно в 09:00:

```javascript
// scripts/daily-routine.sh (bash + python + node)

// 1. Routine на claude.ai генерирует 5 статей из бэклога
// 2. Скрипт делает duplicate-slug guard
// 3. Quality gate (anti-Баден-Баден):
//    - thin content < 800 слов → reject
//    - title overoptimization → reject
//    - keyword stuffing > 1.5% → reject
//    - near-duplicate > 55% совпадений → reject
// 4. typescript typecheck + next build
// 5. git push с retry/rebase
// 6. Vercel deploy + verification по HTTP 200
// 7. IndexNow ping всех новых URL
// 8. Я.Вебмастер recrawl/queue (11 URL/день)
// 9. Y.Метрика + Y.Вебмастер анализ → opportunity miner
// 10. Telegram digest юзеру
```

5 статей в день. Quality gate. Никакой ручной работы. Запустил → работает каждое утро в 9:00 UTC.

### 5. Opportunity Miner — петля обратной связи

Каждое утро после публикации:
- Тянем search-queries из Я.Вебмастера
- Находим запросы на позициях 11-30 («почти топ») → в backlog для дожатия
- URL с показами >50 и CTR <2% → очередь рефреша Title/Description
- Запросы где есть показы но нет страницы → упущенный спрос → backlog
- Сравнение скользящих окон (12 недель истории) → decay-детектор

Это создаёт **самоулучшающуюся систему**: чем больше данных Я.Вебмастера накапливается, тем точнее темы для следующих статей.

### 6. GEO/AIO — попадание в AI-ответы

Из выжимки 19 SEO-видео 2026 года выделили главный сдвиг: **попадание в ответы нейросетей** (Алиса, ChatGPT, Perplexity).

Что сделали:
- `public/llms.txt` — структурированная карта сайта в формате LLMs.txt (новый стандарт)
- `robots.ts` — явно разрешены 11 AI-ботов (GPTBot, PerplexityBot, ClaudeBot, YandexAlice, Google-Extended, CCBot, Applebot-Extended и т.д.)
- FAQPage Schema с 6 Q&A на гео-страницах и блоговых статьях (Алиса любит «вопрос-ответ» формат)
- Структурированный контент в первых 1500 символах каждой статьи

Проверил симуляцией curl с User-Agent от GPTBot/PerplexityBot/ClaudeBot — все 200 OK. Cloudflare не блочит AI-ботов на нашем плане.

### 7. Self-healing dropped URLs

Каждое утро daily-routine проверяет `important-urls` в Я.Вебмастере. Если статья пропала из поиска (NOT_FOUND/REMOVED/DROPPED) — автоматически отправляется на переобход. Это backup на случай когда Яндекс почему-то деиндексирует страницу.

## Что я бы сделал по-другому

1. **Не делать одностраничник на Тильде в первый месяц бизнеса.** Год работы Тильды = год потерянного времени на накопление авторитета домена.

2. **Сразу написать полный список услуг с гео-посадками.** Я начал с 5 услуг → потом достроил 9 регионов. Лучше было сразу 9×3=27 страниц.

3. **Quality gate с самого начала.** У меня в начале пробежало 3-4 тонких статьи. Я их потом дописал, но это было время.

4. **IndexNow подключить в первый день.** Я подключил на третьей неделе. Без IndexNow Яндекс индексирует за 7-14 дней, с IndexNow — за 1-3 дня.

5. **Person markup для E-E-A-T.** Я добавил Person для 3 братьев (Юрий + Валера + Евгений) поздно. Без этого Article author=Organization — это слабее.
