# 🚀 Деплой сайта СК Юрьевич — инструкция для чайника

Пошагово, простыми словами. Делай по порядку, не перепрыгивай.

Домен: **sk-yurievich.ru** · Хостинг: **Vercel** · Код: **GitHub**

---

## ШАГ 1. Купить домен на Reg.ru (10 минут, ~250 ₽/год)

1. Открой https://reg.ru
2. В поиске введи: `sk-yurievich.ru`
3. Если свободен — жми «Зарегистрировать»
4. **Минимальный набор:** только домен. Никаких «защит от спама», «продления автоматически», «хостинга» — это не нужно. Хостинг у нас будет Vercel (бесплатный).
5. Зарегистрируй аккаунт на Reg.ru (email + пароль). Сохрани логин/пароль в надёжное место — они тебе понадобятся раз в год для продления.
6. Оплати картой ~199-299 ₽
7. Получи письмо «Домен зарегистрирован»

Параметры DNS пока не трогаем — настроим на Шаге 4.

## ШАГ 2. Создать GitHub аккаунт + залить код (15 минут)

GitHub — твой «сейф» для кода. Регистрируется бесплатно, навсегда.

1. Открой https://github.com → жми «Sign up»
2. Email, пароль, имя пользователя (например, `yurievich-stroy`). Запомни логин.
3. Подтверди email
4. После регистрации — справа сверху «+» → «New repository»
5. Имя репозитория: `yurievich-site` (английскими)
6. Тип: **Public** (приватные платные)
7. Не ставь галочки «Initialize with README»
8. Жми «Create repository»

Теперь нужно залить наш код. Открой терминал на компьютере и выполни команды (замени `ТВОЙ-ЛОГИН` на реальный):

```bash
cd /home/client2/projects/yurievich-site-v2
git init
git add .
git commit -m "init: SK Yurievich website"
git branch -M main
git remote add origin https://github.com/ТВОЙ-ЛОГИН/yurievich-site.git
git push -u origin main
```

При первом push спросит логин и пароль. Логин — твой GitHub-логин. Пароль — **не** обычный, а Personal Access Token:
- GitHub → справа вверху аватарка → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token (classic)
- Имя любое, срок: 90 дней или больше
- Поставь галочку **repo** (всю)
- Сгенерируй, скопируй токен (он покажется один раз). Это будет твой пароль для git.

После push — обнови страницу репозитория на github.com, увидишь все файлы.

## ШАГ 3. Подключить Vercel (5 минут)

Vercel — бесплатный хостинг для Next.js. Деплоит сам из GitHub за 1 минуту после каждого изменения.

1. Открой https://vercel.com → «Sign Up» → «Continue with GitHub»
2. Разреши доступ Vercel к репозиторию `yurievich-site`
3. На главной — «Add New» → «Project»
4. Найди в списке `yurievich-site` → жми «Import»
5. **Никакие настройки не меняй** — Vercel сам определит Next.js
6. Жми «Deploy»
7. Через 1-2 минуты сайт доступен по адресу типа `yurievich-site-abc123.vercel.app`

Открой URL — это твой рабочий сайт. Через 30 секунд после любого push в GitHub он будет обновляться.

---

## ШАГ 4. Подключить домен sk-yurievich.ru к Vercel (10 минут)

1. На Vercel в своём проекте: Settings → Domains
2. Введи: `sk-yurievich.ru` → Add
3. Vercel покажет 2 DNS-записи которые надо вставить на Reg.ru:
   - Тип A — Value `76.76.21.21`
   - Тип CNAME — Name `www`, Value `cname.vercel-dns.com`

   (точные значения могут отличаться — копируй те, что покажет Vercel)
4. Иди на reg.ru → войди → Мои домены → sk-yurievich.ru → Управление DNS-записями
5. Удали стандартные A-записи (если есть)
6. Добавь две записи из шага 3 (точно как сказал Vercel)
7. Сохрани
8. Подожди 5-60 минут (обычно 10-15)
9. Vercel сам покажет зелёную галочку «Configured»
10. Готово — сайт на https://sk-yurievich.ru

SSL-сертификат Vercel выдаст автоматически (зелёный замочек в браузере) — ничего настраивать не нужно.

## ШАГ 5. SEO: Яндекс.Вебмастер + Google Search Console (10 минут)

Это нужно чтобы Яндекс и Google знали о сайте и начали индексировать.

### Яндекс.Вебмастер
1. https://webmaster.yandex.ru → войди через Яндекс-аккаунт (если нет — заведи)
2. «Добавить сайт» → введи `https://sk-yurievich.ru`
3. Подтверди права. Самый простой способ — **Мета-тег**:
   - Скопируй строку вида `<meta name="yandex-verification" content="abc123..." />`
   - Из неё нужен только код (между `content="` и `"`) — например `abc123...`
   - Пришли его мне в чат, я вставлю в `lib/site.ts` → поле `yandexVerification`
   - Push в GitHub → Vercel задеплоит → подтверди в Вебмастере
4. После подтверждения → «Региональность» → выбери «Санкт-Петербург и Ленинградская область»
5. «Файл Sitemap» → введи: `https://sk-yurievich.ru/sitemap.xml` (автоматически создаётся, всё уже работает)

### Google Search Console
1. https://search.google.com/search-console → войди через Google
2. «Добавить ресурс» → `https://sk-yurievich.ru`
3. Подтверди права через **HTML-тег**:
   - Скопируй код из строки `<meta name="google-site-verification" content="..." />`
   - Пришли мне в чат, я вставлю в `googleVerification`
   - Push → Vercel → подтверди
4. Sitemap → добавь `sitemap.xml`

1. https://metrika.yandex.ru → войди → «Добавить счётчик»
2. Имя счётчика: `СК Юрьевич`
3. Адрес сайта: `sk-yurievich.ru`

5. Настройки → Включить: **Карта кликов**, **Вебвизор**, **Аналитика форм**, **Точный показатель отказа**
6. Создать счётчик. Получишь 8-значный номер вроде `12345678`
7. Пришли его мне → вставлю в `yandexMetrikaId`
8. Push → Vercel → метрика начнёт собирать данные через 24 часа

# 📝 Как редактировать сайт (когда меня нет рядом)

### Способ 1: Прямо в браузере (самый простой)

1. github.com → твой репозиторий → найди файл (например `lib/site.ts` чтобы поменять цены)
2. Жми карандашик в правом верхнем углу
3. Меняй текст
4. Внизу — «Commit changes» (зелёная кнопка)
5. Сайт обновится за 30 секунд

### Способ 2: VSCode на компьютере

1. Скачай VSCode (free) → vscode.dev
2. Установи Git
3. `git clone https://github.com/ТВОЙ-ЛОГИН/yurievich-site.git`
4. Открой папку в VSCode
5. Меняй файлы → сохраняй → `git add . && git commit -m "что поменял" && git push`
6. Сайт обновится за 30 секунд

### Способ 3: Через Claude/ChatGPT в будущем

1. Скачай ZIP из GitHub
2. Открой архив, скажи AI «поменяй такую-то фразу в такой-то секции»
3. AI отдаст изменённые файлы → залей обратно через GitHub в браузере

# 🆘 Что делать если что-то сломалось

| Проблема | Что делать |
|---|---|
| Сайт не открывается | Vercel → Deployments → последний → смотри логи. Если красное «Build failed» — присылай ошибку любому AI, исправит. |
| Домен не привязался | Подожди 60 минут. Проверь DNS на reg.ru — те ли значения. |
| Метрика не работает | Зайди в Yandex.Metrika → проверь что счётчик активен (зелёная точка). Подожди сутки. |
| Забыл пароль GitHub | github.com → «Forgot password» → восстановишь через email |
| Забыл пароль Reg.ru | reg.ru → «Забыли пароль?» → email |
