# СК «Юрьевич» — сайт v2

Продающий сайт для семейной строительной компании (фундаменты + дома под ключ, СПб + ЛО).

Стек: **Next.js 14 (App Router) + Tailwind CSS + TypeScript**.

---

## Что внутри

Главная страница из 12 секций:
1. Hero — заголовок + CTA «Рассчитать стоимость»
2. 4 УТП-иконки (договор / гарантия / братья / 239 объектов)
3. 4 карточки услуг
4. Квиз-калькулятор (5 шагов + контакты)
5. Портфолио (9 объектов)
6. Слайдер отзывов (5 шт)
7. Процесс — 5 шагов
8. Блок про 3 братьев
9. Партнёрство с ЛСР Газобетон
10. FAQ — 8 вопросов
11. Контакты + форма
12. Footer

Плюс:
- API `/api/lead` — приём заявок через Telegram-бота
- Sticky-полоса с телефоном и кнопкой расчёта на мобильных
- Плавающие WhatsApp + Telegram кнопки на десктопе
- SEO мета-теги + Schema.org LocalBusiness
- Страница `/privacy`

## Деплой на Vercel (5 минут)

1. **Зайди на https://vercel.com** и войди через GitHub (или создай аккаунт).
2. **Загрузи код на GitHub:**
   ```bash
   cd /home/client2/projects/yurievich-site-v2
   git init
   git add .
   git commit -m "init"
   # Создай новый репозиторий на github.com → скопируй URL → подставь:
   git remote add origin <URL твоего репо>
   git push -u origin main
   ```
3. **На Vercel** → New Project → выбери репозиторий → Deploy. Никаких настроек менять не надо.
4. Через ~2 минуты сайт доступен по адресу вида `yurievich-site-v2.vercel.app`.
5. **Подключи свой домен** (когда купишь) в Vercel → Settings → Domains.

## Заявки в Telegram

Чтобы заявки приходили в Telegram-бот:

1. Открой Telegram → найди [@BotFather](https://t.me/BotFather) → команда `/newbot` → следуй инструкциям → получи **TOKEN**.
2. Напиши своему боту любое сообщение (хотя бы «привет»).
3. Открой в браузере: `https://api.telegram.org/bot<ТВОЙ_ТОКЕН>/getUpdates`
4. В ответе найди `"chat":{"id": ЧИСЛО}` — это твой **CHAT_ID**.
5. В **Vercel** → Project Settings → Environment Variables добавь:
   - `TELEGRAM_BOT_TOKEN` = твой токен
   - `TELEGRAM_CHAT_ID` = твой chat id
6. Redeploy. Теперь заявки приходят в твой Telegram.

Если переменные не заданы — заявки пишутся в логи сервера (видно в Vercel → Logs).

## Что заменить контентом (плейсхолдеры)

В коде стоят красивые «заглушки», их легко заменить на реальные фото:

- **Hero-фото** — `components/Hero.tsx`, сейчас градиент. Положи реальное фото в `public/images/hero.jpg` и замени блок с `backgroundImage`.
- **Фото братьев** — `components/Brothers.tsx`, сейчас инициалы Ю/В/Е. Положи 3 портрета в `public/images/brothers/` и замени `<div>` на `<Image>`.
- **Портфолио (9 фото)** — `components/Portfolio.tsx`, сейчас иконки + градиенты. Положи фото в `public/images/portfolio/` и замени плейсхолдеры.
- **Лого ЛСР** — `components/LsrBanner.tsx`, сейчас текстовый блок «ЛСР». Положи официальный логотип, если есть юридическое разрешение.

## Что можно подкрутить

- **Цены «от»** — `lib/site.ts` → объект `PRICES`. Сейчас стоят средние по рынку (плита 5500₽/м², ленточный 4800₽/м.п., дом 38000₽/м²). Поправь под реальные.
- **Телефон** — `lib/site.ts` → `SITE.phone`.
- **Тексты отзывов** — `components/Reviews.tsx` → массив `REVIEWS`. Сейчас правдоподобные тексты, но **подмени на реальные с Авито** перед запуском.
- **Объекты в портфолио** — `components/Portfolio.tsx` → массив `OBJECTS`. Подправь типы/локации под реальные кейсы.

3. Настрой цель «отправка формы» на событие `lead_sent` (нужно добавить в `Quiz.tsx` после успешной отправки).
